# Copilot Instructions

## Operating Rules

- Follow the user's requested action unless it would be unsafe or impossible.
- Run concrete checks before speculating about errors.
- Show actual command output when diagnosing failures.
- Do not create new external sessions or resources unless asked.
- If a requested operation fails, report the error and ask before switching approach.
- Do not hide mistakes or continue after a correction without adjusting behavior.

## API And Session Work

- Reuse the session ID supplied by the user.
- Do not create replacement sessions without explicit approval.
- Show the API or CLI response for session operations.
- Treat documentation from the repository or official service docs as the source of truth.

## Code Changes

- Make the requested change.
- Run relevant tests when possible.
- Report files changed and commands run.
- Do not over-expand the scope.

## Red Flags

- Creating multiple resources when one was requested.
- Assuming a command will fail without running it.
- Truncating output when full output was requested.
- Making product or workflow decisions for the user.
- Ignoring explicit user instructions.
